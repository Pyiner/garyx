Garyx Design Review

绑定 Channel 的 StreamEvent 扇出设计

目标是把线程运行产生的 committed StreamEvent，按线程绑定关系统一扇出到所有 channel 端点；gateway/router 不再承担 Markdown、图片、工具调用或平台特殊规则的渲染职责。

核心边界 Gateway/router 只做事件和端点选择；channel 自己消费 StreamEvent 并渲染。
主路径 内置 channel 和新 plugin 都实现 dispatch_stream_event。
兼容路径 旧 subprocess plugin 由 channels 层 adapter 降级到 dispatch_outbound。

1. 目标架构

共享 fanout 是唯一 committed-replay 挂载点。无论 run 从哪里触发，只要有稳定的 thread_id 和 run_id，都进入同一条分发链路。

统一 StreamEvent 扇出

run 来源无关，channel 渲染内聚在 garyx-channels

2. Run 生命周期和绑定变更

run 开始时 snapshot target set。运行中修改 bot/channel 绑定不会影响当前 run；streaming input 追加到同一个 run 时也不会重新选目标。新绑定从下一次 run 开始生效。

Snapshot 语义

稳定性优先，避免一个 run 中途改变投递集合

3. Endpoint Identity 和去重

这里的边界收窄为：gateway/router 只认 channel 给出的 opaque Endpoint Identity。它不关心这个 key 怎么生成、里面有什么字段，也不解析任何 channel 内部字段。

只按 Endpoint Identity 做 equality

生成和语义归属在 channel，gateway 只比较 key 是否相等

4. Dispatcher Contract

事件/字段 设计约束 原因
ToolUse / ToolResult 保持结构化直到 channel renderer Feishu、Telegram、Discord 可以有不同呈现，gateway 不做降级文本
UserAck 只投 origin endpoint，或带 origin Endpoint Identity 让非 origin 忽略 ack 是端点相关事件，不是所有绑定端点都应切分输出
seq 来自 committed per-run replay sequence 用于 replay gap recovery 和 restart reattach 后幂等
delivery outcome 返回 Endpoint Identity、run id、seq、message id、状态 保留 router outbound id persistence 和诊断能力

5. Legacy Plugin Adapter

旧插件兼容降级

adapter 是临时债，不是最终协议

6. 迁移和验收

迁移顺序

  1. 补 opaque Endpoint Identity equality 测试。
  2. channel 侧补 identity 生成契约测试。
  3. 新增 dispatcher-level dispatch_stream_event。
  4. 内置 channel 先接入 native dispatch。
  5. 补 delivery outcome 和 router outbound id 回流。
  6. 对子进程 plugin 做 capability detection 和 legacy adapter。
  7. 所有 run origin 切到共享 fanout。
  8. 删除 gateway 渲染和 channel 特判。

验收重点

  • run 来源无关，所有绑定端点都能收到 committed event。
  • gateway/router 只按 Endpoint Identity equality 去重。
  • 结构化 tool event 不在 gateway 降级。
  • restart reattach 后不重复投递。
  • unsupported structured event 可观测。
